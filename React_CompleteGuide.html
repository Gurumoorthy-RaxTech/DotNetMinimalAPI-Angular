Props vs State — Core Difference
Interview-ல 100% கேப்பாங்க

Props (Properties)

• Parent → Child pass பண்றது
• Read-only (immutable)
• Like method parameters
• Change ஆனா re-render இல்ல 
// Parent sends
<Card name="Guru" />

// Child receives
function Card({ name }) {
  return <h1>{name}</h1>;
}

State

• Component's own data
• Mutable (can change)
• Like private fields
• Change → re-render triggers 
function Counter() {
  const [count, setCount]
    = useState(0);

  return (
    <button
      onClick={() => setCount(count+1)}>
      {count}
    </button>
  );
}
Props = parent-லிருந்து receive பண்றோம் (read-only). State = component-ஓட own data (change பண்ணலாம், UI update ஆகும்). Simple rule: "யாரோட data?" — own-ஆ இருந்தா State, parent-கிட்டிருந்து வந்தா Props.

Virtual DOM — React Speed Secret
Interview must-know concept

How it works:

1. Virtual DOM (Memory copy) → React memory-ல ஒரு lightweight DOM copy வச்சிருக்கும்
2. State change → New Virtual DOM create பண்ணும்
3. Diffing → Old vs New Virtual DOM compare பண்ணும்
4. Reconciliation → Changed parts மட்டும் Real DOM update பண்ணும்
Result → Minimum DOM operations → Fast UI!
Direct DOM manipulation slow. React memory-ல Virtual copy வச்சிருக்கும். Change ஆனா, old vs new compare (diff) பண்ணி, மாறினது மட்டும் update பண்ணும். கடையிலே முழு shelf-ஐ change பண்றது இல்ல, ஒரு item மட்டும் replace பண்றோம்.

api.js — Central Axios Instance
JWT auto-attach + 401 auto-refresh
HttpClient with BaseAddress + DelegatingHandler (auto JWT header) in C#
// src/services/api.js
import axios from 'axios';

const api = axios.create({
  baseURL: 'http://localhost:5260/api/v1',
  headers: { 'Content-Type': 'application/json' },
});

// Request interceptor — every API call-la JWT auto add பண்ணும்
api.interceptors.request.use(config => {
  const token = localStorage.getItem('accessToken');
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

// Response interceptor — 401 வந்தா auto refresh try பண்ணும்
api.interceptors.response.use(
  response => response,
  async error => {
    const original = error.config;
    if (error.response?.status === 401 && !original._retry) {
      original._retry = true;
      try {
        const res = await axios.post('/auth/refresh', {
          accessToken:  localStorage.getItem('accessToken'),
          refreshToken: localStorage.getItem('refreshToken'),
        });
        localStorage.setItem('accessToken', res.data.data.token);
        original.headers.Authorization = `Bearer ${res.data.data.token}`;
        return api(original); // Retry original request
      } catch {
        localStorage.clear();
        window.location.href = '/login';
      }
    }
    return Promise.reject(error);
  }
);

export default api;
interceptors = C#-ல Middleware போல. Request interceptor: API call போற முன்னே JWT add. Response interceptor: 401 வந்தா, refresh token use பண்ணி new token get பண்ணி, original request retry. User-கு logout தெரியாமலே auto handle ஆகும்!

AuthContext.jsx — Global Auth State
Prop drilling avoid — any component-ல user access பண்ணலாம்
C# HttpContext.Session / ClaimsPrincipal — global user info store
// src/context/AuthContext.jsx
import { createContext, useContext, useState, useCallback } from 'react';
import { authService } from '../services/authService';

const AuthContext = createContext(null);

export function AuthProvider({ children }) {
  // Page refresh-லயும் login maintain — localStorage படிக்கிறோம்
  const [user, setUser] = useState(() => {
    const token    = localStorage.getItem('accessToken');
    const username = localStorage.getItem('username');
    const role     = localStorage.getItem('role');
    return token ? { username, role, token } : null;
  });

  const login = useCallback(async (username, password) => {
    const res = await authService.login(username, password);
    const { token, refreshToken, username: uname, role } = res.data.data;
    localStorage.setItem('accessToken',  token);
    localStorage.setItem('refreshToken', refreshToken);
    localStorage.setItem('username',     uname);
    localStorage.setItem('role',          role);
    setUser({ username: uname, role, token });
  }, []);

  const logout = useCallback(async () => {
    try {
      await authService.revoke(localStorage.getItem('refreshToken'));
    } finally {
      localStorage.clear();
      setUser(null);
    }
  }, []);

  return (
    <AuthContext.Provider value={{ user, login, logout, isAdmin: user?.role === 'Admin' }}>
      {children}
    </AuthContext.Provider>
  );
}

// Custom hook — any component-ல useAuth() call பண்ணி user get பண்ணலாம்
export function useAuth() {
  return useContext(AuthContext);
}
AuthProvider = App-ஐ wrap பண்ணுவோம். எந்த child component-லயும் useAuth() call பண்ணி { user, login, logout, isAdmin } get பண்ணலாம். Prop drilling வேண்டாம் — global access!

StudentList.jsx — Main Page
useEffect + useState + API call + Search filter
// src/pages/StudentList.jsx
import { useState, useEffect, useCallback } from 'react';
import { studentService } from '../services/studentService';
import { useAuth } from '../context/AuthContext';
import StudentCard from '../components/StudentCard';

export default function StudentList() {
  const { user, logout, isAdmin } = useAuth();
  const [students, setStudents] = useState([]);
  const [loading,  setLoading]  = useState(true);
  const [error,    setError]    = useState('');
  const [search,   setSearch]   = useState('');

  // useCallback = function memoize பண்ணும் (unnecessary re-create avoid)
  const fetchStudents = useCallback(async () => {
    try {
      setLoading(true);
      const res = await studentService.getAll();
      setStudents(res.data.data); // ApiResponse<List<Student>> → .data.data
    } catch {
      setError('API error — backend running-ஆ check பண்ணுங்க');
    } finally {
      setLoading(false);
    }
  }, []);

  // useEffect = Page_Load / OnGet() — component mount ஆகும்போது run
  useEffect(() => { fetchStudents(); }, [fetchStudents]);

  const handleDelete = async (id) => {
    if (!window.confirm('Delete பண்ணணுமா?')) return;
    await studentService.delete(id);
    // Local state update — API re-call வேண்டாம் (optimistic update)
    setStudents(prev => prev.filter(s => s.id !== id));
  };

  // LINQ .Where() equivalent — search filter
  const filtered = students.filter(s =>
    s.name.toLowerCase().includes(search.toLowerCase()) ||
    s.course.toLowerCase().includes(search.toLowerCase())
  );

  if (loading) return <div>Loading...</div>;
  if (error)   return <div>{error}</div>;

  return (
    <div>
      {filtered.map(student => (
        <StudentCard key={student.id} student={student} onDelete={handleDelete} />
      ))}
    </div>
  );
}
key={student.id} mandatory — React-கு list items track பண்ண. இல்லன்னா warning வரும், performance கெடும். C#-ல Dictionary-ல unique key போல.

useEffect — Side Effects
API calls, subscriptions, timers — component lifecycle
Page_Load / OnGet() / OnInitializedAsync() in Blazor
// Syntax
useEffect(() => {
  // Side effect code
  return () => {
    // Cleanup (optional) — component unmount ஆகும்போது
  };
}, [dependency1, dependency2]); // Dependency array

// ── Pattern 1: On mount once ([] = empty dep) ──
useEffect(() => {
  fetchStudents(); // Page load-லை ஒரே ஒரு தடவை
}, []);

// ── Pattern 2: When dependency changes ──
useEffect(() => {
  fetchById(id); // id change ஆகும்போது run
}, [id]);

// ── Pattern 3: Cleanup (Timer, WebSocket) ──
useEffect(() => {
  const timer = setInterval(fetchLiveData, 5000);
  return () => clearInterval(timer); // Cleanup on unmount
}, []);

// ── Pattern 4: No dependency = every render (usually wrong!) ──
useEffect(() => {
  console.log('Every render'); // ⚠️ Rarely needed
});
Dependency array [] = ஒரே ஒரு தடவை (mount). [id] = id change ஆகும்போது. Empty = every render. Cleanup function = IDisposable.Dispose() போல.

useContext — Global State Access
Prop drilling problem solve பண்ண
// Problem: Prop Drilling — 3 levels down pass பண்றது
<App user={user}>
  <Layout user={user}>       // Layout doesn't need it, just passes
    <Navbar user={user}>     // Finally uses it
    </Navbar></Layout></App>

// Solution: Context — any level direct access
const AuthContext = createContext(null);

// Provide (App level)
<AuthContext.Provider value={{ user, login, logout }}>
  <App />
</AuthContext.Provider>

// Consume (any child — no prop drilling!)
function Navbar() {
  const { user } = useContext(AuthContext);
  return <span>Welcome, {user.username}</span>;
}
Prop drilling = ஒரு data-ஐ A → B → C → D pass பண்றது, B/C use பண்றதில்லன்னாலும். Context-ல A-ல provide பண்ணி D-ல directly consume பண்ணலாம். Like global variable, ஆனா React way.

useMemo & useCallback — Performance
Unnecessary re-computation prevent பண்ண

useMemo — Value cache

// Expensive calculation cache
// C# Lazy<T> போல
const total = useMemo(() =>
  students.reduce(
    (sum, s) => sum + s.fees, 0
  ),
  [students] // Only recalculate when students changes
);

// Search filter cache
const filtered = useMemo(() =>
  students.filter(s =>
    s.name.includes(search)
  ),
  [students, search]
);

useCallback — Function cache

// Function re-create prevent
// Child re-render avoid பண்ண
const handleDelete = useCallback(
  async (id) => {
    await studentService.delete(id);
    setStudents(prev =>
      prev.filter(s => s.id !== id)
    );
  },
  [] // No dependencies = create once
);

// Without useCallback = new function
// every render = child re-renders
useMemo = value cache (calculation). useCallback = function cache. Both dependency array use பண்ணும் — அது change ஆனா மட்டும் recalculate/recreate. Interview: "Performance optimize பண்ண useMemo/useCallback use பண்றேன்" சொல்லுங்க.

Basic Level Fresher / 1yr

What is React? Why use it over plain JavaScript?
React is a UI library by Facebook for building component-based UIs. Benefits: Virtual DOM for performance, reusable components, one-way data flow, large ecosystem. Plain JS-ல DOM manipulation manual-ஆ பண்ணணும் — complex-ஆ ஆகும்போது maintain பண்ண கஷ்டம்.
React = UI library, not full framework. Virtual DOM use பண்றதால் fast. Components-ஆ split பண்றதால் reuse பண்ணலாம்.
What is JSX? Is it mandatory in React?
JSX is a syntax extension for JavaScript that looks like HTML. Babel compiles it to React.createElement() calls. Not mandatory — you can write React.createElement() directly, but JSX makes it readable.
JSX = JavaScript + HTML mixed. Browser இதை directly understand பண்றதில்ல — Babel compile பண்ணி pure JS ஆக்கும். Razor View போல imagine பண்ணுங்க.
Props vs State — difference?
Props: Data passed from parent to child. Read-only (immutable). Like method parameters.
State: Component's own data. Mutable via setState/useState. Change triggers re-render.
Simple rule: "Who owns the data?" — if external = Props, if internal = State.
Props = parent கொடுத்தது (touch பண்றது இல்ல). State = component-ஓட own data (change பண்ணலாம், UI update ஆகும்).
What is Virtual DOM? How does it improve performance?
Virtual DOM is an in-memory lightweight copy of the real DOM. When state changes: (1) New Virtual DOM created, (2) Old vs New diffed (reconciliation), (3) Only changed nodes updated in real DOM. This avoids expensive full DOM re-renders.
React memory-ல copy வச்சிருக்கும். Change ஆனா, diff பார்க்கும், மாறினது மட்டும் update. Full page refresh இல்ல — surgical update. Speed increase இதுதான்.
What is the purpose of the "key" prop in lists?
key helps React identify which items changed, added, or removed in a list. Must be unique among siblings. Using index as key is a bad practice when list items can reorder. Use unique IDs from your data.
key இல்லாம் list render பண்ணா warning வரும். Performance கெடும். student.id போல unique value use பண்ணுங்க, array index-ஐ use பண்றதை தவிருங்க.

Intermediate Level 2–3 yrs

Explain useEffect with examples — when does it run?
useEffect(fn, []) — Runs once on mount (like OnInitialized)
useEffect(fn, [id]) — Runs when id changes
useEffect(fn) — Runs after every render (rarely needed)
Return function = cleanup (runs on unmount or before next effect)
[] = mount-ல ஒரே ஒரு தடவை. [dep] = dep change ஆகும்போது. cleanup function = component போகும்போது (timer clear, subscription unsubscribe).
What is Context API? When to use it vs Props?
Context API provides global state accessible by any component without prop drilling. Use Context for: auth state, theme, language, user preferences.
Don't overuse — frequent updates in Context cause all consumers to re-render. For complex state, consider Zustand or Redux.
Prop drilling = A → B → C → D pass பண்றது B/C don't need. Context-ல A provide, D consume directly. Auth, theme, language போன்ற global things-க்கு use பண்றோம்.
Controlled vs Uncontrolled components?
Controlled: Form data React state-ல manage பண்றோம். value={state} + onChange={handler}. React is "single source of truth".
Uncontrolled: DOM itself manages data. useRef use பண்ணி read பண்றோம். Less React-like.
Production-ல controlled components use பண்றோம். value= state-லிருந்து வரும், onChange= state update பண்ணும். "Single source of truth" — interview answer-ல சொல்லுங்க.
How does JWT authentication work in React?
1. Login → POST credentials → API returns access token + refresh token
2. Store tokens in localStorage (or httpOnly cookie for security)
3. Every API request → add Authorization: Bearer {token} via axios interceptor
4. Token expires → interceptor catches 401 → auto-refresh using refresh token
5. Logout → revoke refresh token → clear localStorage
Axios interceptor use பண்றோம் — manually header add பண்ணது இல்ல. 401 வந்தா auto refresh. உங்கள் project-லேயே இந்த pattern implement பண்ணோம்!
useMemo vs useCallback — difference?
useMemo: Memoizes a computed value. Returns the cached result. Use for expensive calculations (filtering, sorting large arrays).
useCallback: Memoizes a function reference. Returns the cached function. Use to prevent child component re-renders when passing callbacks as props.
useMemo = value cache (sum, filter result). useCallback = function cache (event handlers). Both dependency array use பண்ணும். Premature optimization வேண்டாம் — bottleneck இருந்தா மட்டும் use பண்ணுங்க.

What is React's Reconciliation algorithm?
React's reconciliation (Fiber algorithm) determines the minimum DOM updates needed. Rules: (1) Different element types → destroy and rebuild tree, (2) Same type → update attributes, (3) Lists → use keys to match elements. React 18 uses concurrent rendering — can interrupt, pause, resume rendering.
Fiber = React 16+ reconciler. Work units-ஆ split பண்ணி, priority-based render பண்ணும். High priority (user input) → low priority (background updates) interrupt பண்ணலாம். Smooth UI இதனால்.
Explain React's rendering optimization strategies
1. React.memo — Prevent functional component re-render if props unchanged
2. useMemo — Cache expensive computed values
3. useCallback — Stable function references for child components
4. Code Splitting — React.lazy + Suspense for route-level splitting
5. Virtualization — react-window for rendering only visible list items
6. Key prop — Correct keys prevent unnecessary unmount/remount
Interview-ல கேட்டா 3–4 strategies சொல்லுங்க. React.memo + useCallback combo most common interview answer. "Profiler tool use பண்ணி bottleneck identify பண்றேன்" சொல்லுங்க.
Custom Hook — what is it and when to create one?
A custom hook is a function starting with "use" that reuses stateful logic between components. Extracts useState + useEffect logic into a reusable function — like a service layer for hooks. 
// Custom hook — API fetch logic reuse
function useStudents() {
  const [students, setStudents] = useState([]);
  const [loading,  setLoading]  = useState(true);

  useEffect(() => {
    studentService.getAll().then(res => {
      setStudents(res.data.data);
      setLoading(false);
    });
  }, []);

  return { students, loading };
}

// Use in any component
function StudentList() {
  const { students, loading } = useStudents();
}
Custom hook = logic reuse. Name "use" la start பண்ணணும். Multiple components-ல same API call + loading state logic இருந்தா, custom hook-ல extract பண்ணலாம். DRY principle.
React 18 — Concurrent Features explain பண்ணுங்க
Concurrent Mode: React can pause, interrupt, resume rendering — no blocking.
useTransition: Mark non-urgent state updates (search filter, tab switch) — keeps UI responsive.
useDeferredValue: Defer a value update until urgent work is done.
Automatic Batching: Multiple setState calls batched into one render (even in async).
Suspense for Data: Show fallback while async data loads.
React 18 = React-ஓட biggest update. Concurrent rendering = urgent work (typing) + non-urgent work (filter results) separate பண்றது. UI never freezes. useTransition use பண்ணா search input lag போகும்.
How do you handle state in large React apps?
Local state: useState — component-specific UI state
Shared state: Context API — small to medium apps, infrequent updates
Server state: React Query / TanStack Query — API data, caching, refetching
Global state: Zustand / Redux Toolkit — complex client state
Rule: Start with useState → Context → React Query → Zustand (escalate as needed)
Correct answer: "Server state (API data) = React Query use பண்றேன் — caching, background refetch automatic. Client state = Zustand simple-ஆ இருக்கு, Redux verbosity வேண்டாம்". Interview-ல இந்த hierarchy சொல்லுங்க.
Code Splitting and Lazy Loading in React?
React.lazy() + Suspense — load component only when needed (route-level splitting). Reduces initial bundle size. Webpack/Vite automatically splits at lazy boundaries. 
// Without lazy — all JS upfront load ஆகும்
import StudentList from './pages/StudentList';

// With lazy — only when user visits /students
const StudentList = React.lazy(
  () => import('./pages/StudentList')
);

<Suspense fallback={<div>Loading...</div>}>
  <StudentList />
</Suspense>
Large apps-ல initial load slow ஆகும். Lazy loading use பண்ணா, user click பண்ணும்போது மட்டும் JS load ஆகும். First paint fast. Resume போடும்போது "Code splitting + lazy loading implement பண்ணினேன்" சொல்லுங்க.
