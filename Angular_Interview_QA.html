Angular என்ன? — Architecture Overview Basics

What is Angular?

Angular is a TypeScript-based front-end framework by Google for building Single Page Applications (SPA). It handles routing, forms, HTTP calls, and state — everything in one framework.

Google create panniya TypeScript framework. React mathiri library illai — full framework! SPA (Single Page Application) build pannuvom — oru HTML page thaan, URL maarina content dynamically change aagum, full page reload illai.

Angular Architecture — 6 Main Building Blocks

Angular app consists of: Components → Templates → Services → Modules (old) → Routing → Directives

// Angular Application Structure
AppComponent (Root)
  ├── NavbarComponent
  ├── LoginComponent  ← Route: /login
  ├── DashboardComponent ← Route: /dashboard
  │     ├── StatsCardComponent (child)
  │     └── ChartComponent (child)
  └── StudentListComponent ← Route: /students

// Each Component has 3 parts:
student.component.ts    // Logic (TypeScript class)
student.component.html  // Template (HTML)
student.component.scss  // Styles (CSS/SCSS)
Angular app = Component tree. Oru root component irukum (AppComponent), adhu keezhaye child components irukum. Har component ku 3 files — logic (.ts), HTML (.html), styles (.scss).
Angular vs React vs Vue — difference?
Angular: Full framework (Google), TypeScript mandatory, opinionated — has routing, forms, HTTP built-in.
React: UI library (Meta), JavaScript/TypeScript, flexible — need extra packages for routing, state.
Vue: Progressive framework, gentle learning curve, popular in Asia.
For .NET projects — Angular is most common because both are strongly typed.
Angular = Full framework, TypeScript, Google. React = UI library, Facebook. .NET projects la Angular prefer pannuvanga — both TypeScript + C# strongly typed, similar concepts.

Component — Angular UI Building Block Basics

Component Anatomy

Every UI piece in Angular is a Component. It has a TypeScript class with @Component decorator.

// student.component.ts
import { Component, OnInit, Input, Output, EventEmitter } from '@angular/core';

@Component({
  selector: 'app-student',   // HTML la <app-student> use pannuvom
  standalone: true,           // Modern Angular — no NgModule needed
  templateUrl: './student.component.html',
  styleUrls: ['./student.component.scss']
})
export class StudentComponent implements OnInit {

  // @Input — Parent → Child data pass pannuvom
  @Input() studentName: string = '';

  // @Output — Child → Parent event emit pannuvom
  @Output() deleted = new EventEmitter<number>();

  title = 'Student List';
  students: string[] = ['Guru', 'Basha', 'Sridhar'];

  // ngOnInit — component load aana first thing run aagum
  ngOnInit(): void {
    console.log('Component loaded!');
  }

  deleteStudent(id: number): void {
    this.deleted.emit(id); // Parent ku event notify pannuvom
  }
}
<!-- student.component.html -->
<h2>{{ title }}</h2>          <!-- Interpolation -->

<p>Hello, {{ studentName }}!</p>  <!-- @Input value display -->

<ul>
  <li *ngFor="let s of students">
    {{ s }}
    <button (click)="deleteStudent(1)">Delete</button>
  </li>
</ul>

<!-- Parent component la use pannuvom: -->
<app-student
  [studentName]="'Gurumoorthy'"
  (deleted)="onStudentDeleted($event)"
></app-student>
Component = Angular la oru UI block. @Component decorator use pannuvom. @Input = parent kooda data receive pannuvom. @Output + EventEmitter = parent ku event send pannuvom. ngOnInit = component create aana first API call, data load panna use pannuvom.
@Input and @Output என்ன? Real world example?
@Input: Parent → Child la data pass pannuvom. Like function parameter.
@Output + EventEmitter: Child → Parent ku event notify pannuvom. Like callback.
Example: StudentList (parent) → StudentCard (child). Card la delete button click pannuvom → parent ku event emit → parent list update pannuvom.
@Input = parent irunthu data vangarom. @Output = child irunthu parent ku event anupuvom. Real world: Product list parent → Product card child. Card la add-to-cart click → parent notify → cart update.

Data Binding — 4 Types Basics

4 Types of Data Binding

// 1. INTERPOLATION — {{ }} — Component → Template (One-way)
<h1>{{ title }}</h1>
<p>Total: {{ students.length }}</p>
<p>{{ 2 + 2 }}</p>  <!-- Expressions also work -->

// 2. PROPERTY BINDING — [] — Component → Template (One-way)
<img [src]="imageUrl">            <!-- src = component variable -->
<button [disabled]="isLoading">Save</button>
<input [value]="username">

// 3. EVENT BINDING — () — Template → Component (One-way)
<button (click)="saveStudent()">Save</button>
<input (input)="onSearch($event)">
<form (submit)="onSubmit()">

// 4. TWO-WAY BINDING — [(ngModel)] — Both directions
// Import: FormsModule in component imports
<input [(ngModel)]="searchText">
<p>You typed: {{ searchText }}</p>
// Input change → searchText update → p tag auto update
{{ }} = class variable la irukuradha HTML la show pannuvom. [ ] = HTML attribute ku value bind pannuvom (disabled, src, etc). ( ) = button click, input change mathiri events catch pannuvom. [( )] = input field change aana class variable change aagum, class variable change aana input field change aagum — two-way!
Two-way binding epdi work aagum? [(ngModel)] banana brackets enna?
Banana in a box! [()] = Property binding + Event binding combined.
[(ngModel)]="name" is shorthand for [ngModel]="name" (ngModelChange)="name=$event"
When user types → ngModelChange fires → updates name property → DOM updates automatically.
[(ngModel)] = "banana in a box" — [] property binding + () event binding combine pannuvom. Input type panna → component variable update → everywhere auto update. FormsModule import panna maranthal error varum!

Directives — ngIf, ngFor, ngClass Basics

<!-- *ngIf — Conditional render -->
<div *ngIf="isLoggedIn">Welcome Admin!</div>
<div *ngIf="isLoading; else content">Loading...</div>
<ng-template #content>Data loaded!</ng-template>

<!-- Modern Angular 17+ — @if @else (no *ngIf needed) -->

  <div>Welcome!</div>
} @else {
  <div>Please login</div>
}

<!-- *ngFor — Loop -->
<tr *ngFor="let student of students; let i = index; trackBy: trackById">
  <td>{{ i + 1 }}</td>
  <td>{{ student.name }}</td>
</tr>

<!-- Modern Angular 17+ — @for -->
@for (student of students; track student.id) {
  <tr><td>{{ student.name }}</td></tr>
}

<!-- ngClass — Dynamic CSS class -->
<div [ngClass]="{'active': isActive, 'error': hasError}"></div>
<button [class.disabled]="isLoading">Save</button>

<!-- ngStyle — Dynamic styles -->
<div [ngStyle]="{'color': isError ? 'red' : 'green', 'font-size': '16px'}"></div>

<!-- ngSwitch -->
<div [ngSwitch]="role">
  <span *ngSwitchCase="'Admin'">🔴 Admin</span>
  <span *ngSwitchCase="'Student'">🟢 Student</span>
  <span *ngSwitchDefault>Unknown</span>
</div>
*ngIf = if condition true aana HTML show pannuvom. *ngFor = array loop pannavom (C# foreach mathiri). ngClass = condition based la CSS class add/remove pannuvom. trackBy use pannuvom — performance improve aagum (har loop la new DOM create panna vendama track pannuvom).
trackBy en use pannuvom *ngFor la?
Without trackBy, Angular destroys and recreates ALL DOM elements when data changes. With trackBy, it identifies items by unique key (like id) and only updates changed items — big performance boost for large lists.
trackBy illama = list change aana ALL elements delete → recreate = slow. trackBy kooda = only changed element update = fast. 100 students irundha, oru student update aana 99 DOM elements touch pannama irukum!

Services & Dependency Injection Basics

Service — Shared Business Logic

Services hold business logic, API calls, shared data. Injected into components via DI — same as ASP.NET Core's DI!

Service = reusable logic. Student list 5 components la use pananumna, 5 lom same code ezhudhathe — oru Service la ezhudhi inject pannuvom. .NET la AddScoped/AddSingleton mathiri — Angular la providedIn: 'root' = Singleton.
// student.service.ts
import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable({
  providedIn: 'root'  // App-level Singleton — everywhere inject panlam
               // 'component' = new instance per component
               // [] = specific module only
})
export class StudentService {

  private apiUrl = 'http://localhost:5260/api/v2/students';

  constructor(private http: HttpClient) {}  // HttpClient inject aagum

  getAll(): Observable<Student[]> {
    return this.http.get<Student[]>(this.apiUrl);
  }
}

// component la inject pannuvom
export class StudentListComponent {
  constructor(private studentService: StudentService) {}
  // OR modern way:
  private studentService = inject(StudentService);
}
providedIn: 'root' vs providers array — difference?
providedIn: 'root' = App-wide singleton, tree-shakeable (unused service removed in build).
providers: [StudentService] in component = New instance for that component and children.
Use 'root' for shared services (auth, student). Use component-level for isolated state.
providedIn: 'root' = Singleton, oru instance app full la. Component la providers = har component ku new instance. .NET la Singleton vs Scoped mathiri concept same thaan!

HttpClient — API Calls Basics

// Component la HTTP calls
@Component({ ... })
export class StudentListComponent implements OnInit {

  students: Student[] = [];
  isLoading = false;
  error = '';

  private studentService = inject(StudentService);

  ngOnInit() {
    this.loadStudents();
  }

  loadStudents() {
    this.isLoading = true;

    this.studentService.getAll().subscribe({
      next: (data) => {         // Success — data vanduchu
        this.students = data;
        this.isLoading = false;
      },
      error: (err) => {         // Error — network fail, 500 etc
        this.error = err.message;
        this.isLoading = false;
      },
      complete: () => {         // Observable complete aagiduthu
        console.log('Done');
      }
    });
  }

  createStudent(student: Student) {
    this.studentService.create(student).subscribe({
      next: () => this.loadStudents(),  // List refresh pannuvom
      error: (err) => alert('Error: ' + err.message)
    });
  }
}
HttpClient = .NET la HttpClient mathiri thaan. get(), post(), put(), delete() methods irukku. Observable return pannuvom — subscribe() panna vendiyathu. next = success, error = fail, complete = done. Async operation — result vandhum subscribe callback la varum.

Routing & Navigation Intermediate

// app.routes.ts
import { Routes } from '@angular/router';

export const routes: Routes = [
  { path: '', redirectTo: '/dashboard', pathMatch: 'full' },
  { path: 'login', loadComponent: () =>
      import('./login/login.component').then(m => m.LoginComponent) },
  { path: 'dashboard', loadComponent: () =>
      import('./dashboard/dashboard.component').then(m => m.DashboardComponent),
    canActivate: [authGuard] },           // Guard protect pannuvom
  { path: 'students/:id', component: StudentDetailComponent }, // Route param
  { path: '**', component: NotFoundComponent }  // Wildcard 404
];

// Navigation — 2 ways
// 1. HTML Template la:
<a [routerLink]="['/students', student.id]">View</a>
<a routerLink="/dashboard" routerLinkActive="active">Dashboard</a>

// 2. Component la programmatic:
private router = inject(Router);
this.router.navigate(['/students', id]);
this.router.navigateByUrl('/login');

// Route params read pannuvom
private route = inject(ActivatedRoute);
const id = this.route.snapshot.paramMap.get('id');       // One-time
this.route.params.subscribe(p => this.id = p['id']);       // Reactive

// Query params: /students?page=2&search=guru
const page = this.route.snapshot.queryParamMap.get('page');
loadComponent = Lazy loading — page open agum pothu thaan component download aagum. First load fast aagum. canActivate guard = auth check pannuvom — login illama dashboard access panna mudiyathu. routerLink = href mathiri, aana full page reload illai.
Lazy Loading என்ன? En use pannuvom?
Without lazy loading, Angular bundles ALL components into one JS file — initial load is slow.
With lazy loading (loadComponent), each route's code is split into separate chunks and downloaded only when the user navigates to that route. Faster initial load!
Lazy loading illama = 1 JS file la all components = 5MB download = slow. Lazy loading = har route ku separate chunk = only needed route download = fast. Real project la mandatory use pannuvanga.

// REACTIVE FORMS — Component la
import { FormBuilder, FormGroup, Validators, ReactiveFormsModule } from '@angular/forms';

export class StudentFormComponent {
  private fb = inject(FormBuilder);

  studentForm: FormGroup = this.fb.group({
    name:   ['', [Validators.required, Validators.minLength(3)]],
    email:  ['', [Validators.required, Validators.email]],
    age:    [null, [Validators.required, Validators.min(10), Validators.max(100)]],
    course: ['', Validators.required]
  });

  // Form controls access pannuvom
  get nameControl() { return this.studentForm.get('name'); }

  onSubmit() {
    if (this.studentForm.invalid) return;  // Validation fail = stop
    const data = this.studentForm.value;   // All form values object
    console.log(data); // { name: 'Guru', email: 'guru@...', age: 25, course: 'CS' }
  }

  resetForm() { this.studentForm.reset(); }
}

<!-- HTML Template -->
<form [formGroup]="studentForm" (ngSubmit)="onSubmit()">

  <input formControlName="name" placeholder="Name">
  <!-- Error messages -->
  @if (nameControl?.invalid && nameControl?.touched) {
    @if (nameControl?.errors?.['required']) {
      <span class="error">Name required</span>
    }
    @if (nameControl?.errors?.['minlength']) {
      <span class="error">Min 3 characters</span>
    }
  }

  <input formControlName="email" type="email">
  <button type="submit" [disabled]="studentForm.invalid">Save</button>
</form>
Reactive forms = TypeScript la form create pannuvom, HTML la connect pannuvom. Validators = required, email, minLength, min, max built-in irukku. Custom validator ezhudhalaam. Form submit panna invalid aana button disable pannuvom. formGroup.value = all input values object aa varum.

RxJS & Observables Intermediate

Observable — Async data streams

Observable is like a Promise but can emit multiple values over time. Used for HTTP, events, state. RxJS = Reactive Extensions for JavaScript.

Observable = Promise la difference ithanai thaan: Promise oru value return pannuvom, Observable multiple values emit pannuvom over time. HTTP call, timer, user events — ellaam Observable. subscribe() panna mattum data varum.
// Most used RxJS Operators
import { map, filter, tap, switchMap, debounceTime,
         distinctUntilChanged, catchError, takeUntil, forkJoin } from 'rxjs/operators';

// map — data transform pannuvom (array.map mathiri)
this.http.get<ApiResponse>('/students').pipe(
  map(response => response.data),  // response.data extract pannuvom
  tap(data => console.log(data))   // Debug — data change panna vendaam
).subscribe(students => this.students = students);

// switchMap — search autocomplete (cancel previous request)
searchControl.valueChanges.pipe(
  debounceTime(300),          // 300ms wait — every keypress la API call illai
  distinctUntilChanged(),     // Same value again type panna API call skip
  switchMap(term =>           // New search → old request cancel → new request
    this.studentService.search(term)
  )
).subscribe(results => this.results = results);

// forkJoin — Multiple API calls parallel (all complete aana callback)
forkJoin({
  students: this.studentService.getAll(),
  courses:  this.courseService.getAll()
}).subscribe(({ students, courses }) => {
  this.students = students;
  this.courses = courses;
});

// BehaviorSubject — Shared state (login status, cart count etc)
private isLoggedInSubject = new BehaviorSubject<boolean>(false);
isLoggedIn$ = this.isLoggedInSubject.asObservable(); // Public read-only
this.isLoggedInSubject.next(true); // Update pannuvom
debounceTime(300) = user type panna 300ms wait panni then API call — har key press la API call pannama. switchMap = old request cancel pannuvom new one start — search la important. BehaviorSubject = current value store panni new subscribers ku immediately emit pannuvom.
switchMap vs mergeMap vs concatMap — difference?
switchMap: New value varum pothu previous Observable cancel. (Search — user new letter type panna old search cancel)
mergeMap: All Observables parallel run. Cancel panna mattom. (Multiple independent API calls)
concatMap: One by one, queue la. Previous complete aana next start. (Order important operations)
switchMap = search box la use pannuvom — new type panna old request cancel. mergeMap = parallel calls. concatMap = one by one — payments, ordered operations ku use pannuvom.

// Custom Pipe — Search filter
@Pipe({ name: 'searchFilter', standalone: true })
export class SearchFilterPipe implements PipeTransform {
  transform(students: Student[], search: string): Student[] {
    if (!search) return students;
    return students.filter(s =>
      s.name.toLowerCase().includes(search.toLowerCase())
    );
  }
}

Lifecycle Hooks Intermediate

export class MyComponent implements OnInit, OnDestroy, OnChanges {

  // 1. constructor — Angular DI inject panna first call
  // API calls panna vendaam — too early
  constructor(private service: StudentService) {}

  // 2. ngOnChanges — @Input values change aagum pothu call aagum
  ngOnChanges(changes: SimpleChanges) {
    if (changes['studentId']) {
      this.loadStudent(this.studentId); // New ID varum pothu reload
    }
  }

  // 3. ngOnInit — Component ready! API calls, data load here
  ngOnInit() {
    this.loadData();  // ✅ Best place for API calls
  }

  // 4. ngAfterViewInit — DOM render complete, ViewChild available
  ngAfterViewInit() {
    // chart.js, video player initialize — DOM need pandra libraries
  }

  // 5. ngOnDestroy — Component remove aagum pothu cleanup
  ngOnDestroy() {
    this.subscription?.unsubscribe();  // Memory leak prevent!
    this.signalRService.stopConnection();
  }
}
ngOnInit = component ready aana call aagum — API calls idhe podu. ngOnDestroy = component destroy aagum pothu — subscriptions unsubscribe panna marandha memory leak varum! Interview la memory leak question kepalanga.
Constructor vs ngOnInit — which to use for API calls?
Always use ngOnInit for API calls, not constructor.
Constructor = DI (Dependency Injection) only. Angular hasn't fully initialized the component yet — @Input values not available.
ngOnInit = Component fully initialized, @Input values received, DOM ready to use.
Constructor la @Input values illai — Angular inject panna aachu but initialization illai. ngOnInit la @Input ready, DOM ready, safe aa API call pannum. Interview la itha direct kepalanga!

Guards — Route Protection Intermediate

// auth.guard.ts — Functional Guard (Angular 14+)
import { inject } from '@angular/core';
import { CanActivateFn, Router } from '@angular/router';
import { AuthService } from '../services/auth';

export const authGuard: CanActivateFn = (route, state) => {
  const authService = inject(AuthService);
  const router = inject(Router);

  if (authService.hasToken()) {
    return true;  // ✅ Access allowed
  }

  // Login page ku redirect pannuvom
  router.navigate(['/login'], {
    queryParams: { returnUrl: state.url }  // Login aana redirect back
  });
  return false;  // ❌ Access denied
};

// Role-based guard
export const adminGuard: CanActivateFn = () => {
  const auth = inject(AuthService);
  return auth.isAdmin() ? true : inject(Router).createUrlTree(['/forbidden']);
};

// Routes la use pannuvom
{ path: 'dashboard', component: DashboardComponent, canActivate: [authGuard] },
{ path: 'admin',     component: AdminComponent,     canActivate: [authGuard, adminGuard] }
Guard = route access panna before check pannuvom. authGuard = token irukka check → illana login redirect. adminGuard = Admin role check. Multiple guards possible — ellaam pass aana thaan route access. .NET la [Authorize] attribute mathiri concept.

Interceptors — HTTP Middleware Intermediate

Interceptors sit between every HTTP request/response. Use for: attach JWT token, global error handling, loading spinner, logging.

.NET la Middleware mathiri — har HTTP request/response la intercept pannuvom. JWT token attach, 401 handle, loading spinner show/hide — centralized location la pannuvom. Har component la panna vendaam.
// jwt-interceptor.ts — Our project la irukku
export const jwtInterceptor: HttpInterceptorFn = (req, next) => {
  const token = inject(AuthService).getToken();

  // Token attach pannuvom
  const authReq = token
    ? req.clone({ setHeaders: { Authorization: `Bearer ${token}` } })
    : req;

  return next(authReq).pipe(
    catchError((error: HttpErrorResponse) => {
      if (error.status === 401) {
        // Auto refresh token
      }
      return throwError(() => error);
    })
  );
};

// Loading interceptor — all requests la spinner show
export const loadingInterceptor: HttpInterceptorFn = (req, next) => {
  const loading = inject(LoadingService);
  loading.show();
  return next(req).pipe(
    finalize(() => loading.hide())  // Success or error — hide pannuvom
  );
};

// app.config.ts la register pannuvom
provideHttpClient(withInterceptors([jwtInterceptor, loadingInterceptor]))

Signals (Angular 17+) — Modern State Management Advanced

Signals are Angular's new reactive primitive — simpler than RxJS for component state. No subscribe(), no memory leaks, auto-tracks dependencies.

Signals = new way to manage state. BehaviorSubject mathiri but simpler. subscribe() vendaam, unsubscribe() vendaam — automatic. Angular 17+ projects la signals use pannuvanga.
import { signal, computed, effect } from '@angular/core';

export class CounterComponent {

  // signal() — writable reactive value
  count = signal(0);
  name  = signal('Guru');

  // computed() — auto-calculated, updates when dependencies change
  doubled = computed(() => this.count() * 2);
  greeting = computed(() => `Hello ${this.name()}!`);

  // effect() — side effects when signal changes
  constructor() {
    effect(() => {
      console.log('Count changed to:', this.count());
    });
  }

  increment() { this.count.update(v => v + 1); }
  setName(n: string) { this.name.set(n); }
}

<!-- Template la use pannuvom — signal() call panna value varum -->
<p>Count: {{ count() }}</p>         <!-- Note: () call panna vendiyathu -->
<p>Doubled: {{ doubled() }}</p>
<button (click)="increment()">+1</button>
